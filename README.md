# Rockman-Style Action Game Logic Prototype
Unity を使用した 2D アクションゲームの「挙動・ロジック」を重視したプロトタイプです。
マップデザインよりも、キャラクター操作の気持ちよさや物理的な反応をスクリプトでどう表現するかにフォーカスしています。

## 🔍 このプロジェクトで試していること
- 慣性・空中制御・二段ジャンプ・ダッシュを組み合わせた、軽快なプレイヤー操作
- ノックバック・死亡演出を含む敵キャラクターのリアクション
- シンプルなビーム射撃とヒット演出
- パララックスを用いた 2D カメラ演出

## 🎮 操作方法
アクション	操作キー	補足
左右移動	← / →（または A / D）	Unity 標準の Horizontal 軸
ジャンプ	Z	接地時ジャンプ / 空中で二段ジャンプ
ダッシュ	X	一定時間だけ前方に高速移動
ビーム攻撃	C	正面方向にビームを発射
※ 入力処理は Movement.cs、物理挙動は PlayerController.cs が担当しています。

## 🚀 プレイヤー制御（robo）
プレイヤーの入力と物理挙動を、役割ごとにクラス分割しています。
- **'Attack.cs'**
プレイヤーのビーム攻撃を担当。C キー入力時に攻撃可能かを判定し、ビームプレハブを攻撃位置に生成して発射します。キャラクターの向きからビームの進行方向を決定し、クールタイムで連射を制御します。

- **'BeamShot.cs'**
発射されたビーム弾そのものの挙動を制御。指定方向へ一定速度で移動させ、一定距離を超えると自動破棄します。何かに衝突するとヒット位置に移動してヒットアニメーションを再生し、敵に対してはダメージ処理（ApplyDamage 呼び出し）を行った後、少し経ってから自身を破棄します。

- **'Movement.cs'**
キーボード入力を読み取り、PlayerController に「どの方向へ動きたいか」「ジャンプ／ダッシュしたいか」を渡す入力レイヤーです。左右入力を移動速度に変換し、Z キーでジャンプ、X キーでダッシュ要求フラグを立てます。アニメーターの Speed パラメータもここで更新します。

- **'PlayerController.cs'**
プレイヤーの本格的な物理挙動を制御する中核クラスです。接地判定や壁判定を行い、地上と空中で異なる減速パラメータを用いてスムーズな加速・減速を実現します。
一段ジャンプ＋空中で一度だけ可能な二段ジャンプ、一定時間だけ高速で前進するダッシュ、最大落下速度の制限などをまとめて管理し、速度や状態に応じてアニメーションパラメータ（ジャンプ中・落下中・着地など）を切り替えます。

## 🤖 敵 AI と物理レスポンス（Enemy）
敵キャラクターごとに、役割の異なるシンプルな挙動を実装しています。

- **'BeamShoe_enemy.cs'**
敵のビーム攻撃用として用意されたスクリプト。現在は雛形のみで、今後ビーム発射ロジックを実装する予定のクラスです。

- **'droneEnemy.cs'**
水平方向に往復移動するドローン型の敵です。初期位置を中心に、指定距離内を行ったり来たりし、端に到達すると向きを反転します。
ダメージを受けるとライフを減少させ、残っている場合はノックバックさせたあと短いクールタイムを挟んで移動を再開します。ライフが 0 以下になると死亡フラグを立て、コライダー無効化 → 死亡アニメーション再生 → 一定時間後に削除、という一連の処理を行います。

- **'turrentEnemy.cs'**
固定砲台（タレット）タイプの敵です。一定間隔ごとに攻撃ポイントからビームプレハブを生成し、自身の向きに合わせて発射します。
ダメージを受けるとライフを減らし、ライフが 0 以下になると削除されます。アニメーター参照も持っており、将来的に発射・被弾・破壊アニメーションと連動させることを想定しています。

## 🎥 カメラシステム（camera）
- **'camaraController.cs'**
プレイヤーを追従するカメラと、手前の背景レイヤーのパララックス効果を制御します。
前フレームからのカメラ移動量を元に背景オブジェクトを少しだけ遅れて動かすことで、奥行き感のあるスクロールを表現します。StopFollow フラグにより、イベント時などにパララックスを一時停止させることも可能です。

## 🎞 アニメーション
市販のドット絵アセットをベースに、Unity の Animation / Animator を用いて各種モーション（待機・移動・ジャンプ・攻撃・被弾・死亡など）を構成しています。
スクリプト側では、速度・接地状態・攻撃状態などのゲームロジックに応じてアニメーションパラメータを切り替える設計になっています。
